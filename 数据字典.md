# Online IDE 数据字典

本文档整理项目中的核心数据结构，覆盖 MongoDB 集合、字段含义、枚举值、实体关系和主要 API 数据对象。

## 1. 数据库概览

数据库：`online-ide`

项目使用 MongoDB + Mongoose，主要集合如下：

| 集合 | Mongoose 模型 | 定义文件 | 作用 |
| --- | --- | --- | --- |
| `users` | `User` | `backend/src/models/User.js` | 保存老师、学生账号、角色和权限 |
| `files` | `File` | `backend/src/models/File.js` | 保存用户个人代码文件 |
| `submissions` | `Submission` | `backend/src/models/Submission.js` | 保存学生提交给老师的代码 |
| `systemsettings` | `SystemSetting` | `backend/src/models/SystemSetting.js` | 保存系统级配置，例如 AI API 配置 |

说明：`backend/src/models/userModel.js` 只是兼容导出，实际指向 `User.js`。

## 2. 通用字段

Mongoose 默认字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | `ObjectId` | MongoDB 自动生成的主键 |
| `createdAt` | `Date` | 创建时间，由 `timestamps: true` 自动维护 |
| `updatedAt` | `Date` | 更新时间，由 `timestamps: true` 自动维护 |

通用响应字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 请求是否成功 |
| `error` | `String` | 请求失败时的错误信息 |
| `message` | `String` | 部分成功操作返回的提示信息 |

## 3. 枚举值

### 3.1 用户角色 `role`

| 值 | 中文含义 | 说明 |
| --- | --- | --- |
| `teacher` | 老师 | 可管理用户、查看学生提交，默认不能提交代码 |
| `student` | 学生 | 可运行、保存、提交代码，默认不能管理用户 |

### 3.2 编程语言 `language`

| 值 | 中文含义 | 文件后缀 | 运行方式 |
| --- | --- | --- | --- |
| `python` | Python | `.py` | `python3 main.py` |
| `cpp` | C++ | `.cpp` | `g++` 编译后运行 |
| `nodejs` | Node.js | `.js` | `node main.js` |

### 3.3 AI 服务商 `provider`

| 值 | 中文含义 | 默认 API 地址 | 默认模型 |
| --- | --- | --- | --- |
| `siliconflow` | 硅基流动 | `https://api.siliconflow.cn/v1` | `deepseek-ai/DeepSeek-V3` |
| `openai` | OpenAI 兼容接口 | `https://api.openai.com/v1` | `gpt-4o-mini` |
| `custom` | 自定义兼容接口 | 用户填写 | 用户填写 |

## 4. 集合：users

用途：保存老师和学生账号。

模型文件：`backend/src/models/User.js`

| 字段 | 类型 | 必填 | 默认值 | 约束 / 索引 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `_id` | `ObjectId` | 是 | 自动生成 | 主键 | 用户唯一 ID |
| `username` | `String` | 是 | 无 | `unique: true`，`trim: true`，`lowercase: true` | 登录用户名，统一转小写 |
| `name` | `String` | 否 | `''` | 无 | 显示名；为空时会在校验前填充为 `username` |
| `passwordHash` | `String` | 是 | 无 | `select: false` | 密码哈希，默认查询不返回 |
| `role` | `String` | 否 | `student` | 枚举：`teacher` / `student` | 用户角色 |
| `permissions.canRunCode` | `Boolean` | 否 | `false` | 无 | 是否允许运行代码 |
| `permissions.canSaveFiles` | `Boolean` | 否 | `false` | 无 | 是否允许保存和删除个人文件 |
| `permissions.canSubmitCode` | `Boolean` | 否 | `false` | 无 | 是否允许提交代码给老师 |
| `permissions.canManageUsers` | `Boolean` | 否 | `false` | 无 | 是否允许管理用户 |
| `permissions.canManageAiSettings` | `Boolean` | 否 | `false` | 无 | 是否允许配置系统级 AI |
| `permissions.canReviewSubmissions` | `Boolean` | 否 | `false` | 无 | 是否允许查看学生提交 |
| `isActive` | `Boolean` | 否 | `true` | 无 | 账号是否启用 |
| `createdAt` | `Date` | 是 | 自动生成 | 无 | 创建时间 |
| `updatedAt` | `Date` | 是 | 自动生成 | 无 | 更新时间 |

### 4.1 默认权限

| 权限 | 老师默认值 | 学生默认值 |
| --- | --- | --- |
| `canRunCode` | `true` | `true` |
| `canSaveFiles` | `true` | `true` |
| `canSubmitCode` | `false` | `true` |
| `canManageUsers` | `true` | `false` |
| `canManageAiSettings` | `true` | `false` |
| `canReviewSubmissions` | `true` | `false` |

### 4.2 对外公开用户对象

`toPublicJSON()` 返回字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `_id` | `ObjectId` | 用户 ID |
| `username` | `String` | 用户名 |
| `name` | `String` | 显示名 |
| `role` | `String` | 角色 |
| `permissions` | `Object` | 权限对象 |
| `isActive` | `Boolean` | 是否启用 |
| `createdAt` | `Date` | 创建时间 |
| `updatedAt` | `Date` | 更新时间 |

不返回：`passwordHash`。

## 5. 集合：files

用途：保存当前用户的个人代码文件。

模型文件：`backend/src/models/File.js`

| 字段 | 类型 | 必填 | 默认值 | 约束 / 索引 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `_id` | `ObjectId` | 是 | 自动生成 | 主键 | 文件 ID |
| `owner` | `ObjectId` | 是 | 无 | `ref: User`，`index: true` | 文件所属用户 |
| `name` | `String` | 是 | 无 | 无 | 文件名 |
| `content` | `String` | 否 | `''` | 无 | 文件内容，即代码文本 |
| `language` | `String` | 是 | 无 | 枚举：`python` / `cpp` / `nodejs` | 文件语言 |
| `createdAt` | `Date` | 是 | 自动生成 | 无 | 创建时间 |
| `updatedAt` | `Date` | 是 | 自动生成 | 无 | 更新时间 |

访问规则：

| 操作 | 限制 |
| --- | --- |
| 查询文件 | 只能查询 `owner` 为当前登录用户的文件 |
| 创建 / 更新 / 删除 | 需要 `canSaveFiles` 权限 |

## 6. 集合：submissions

用途：保存学生提交给老师的代码记录。

模型文件：`backend/src/models/Submission.js`

| 字段 | 类型 | 必填 | 默认值 | 约束 / 索引 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `_id` | `ObjectId` | 是 | 自动生成 | 主键 | 提交记录 ID |
| `student` | `ObjectId` | 是 | 无 | `ref: User`，`index: true` | 提交代码的学生 |
| `teacher` | `ObjectId` | 是 | 无 | `ref: User`，`index: true` | 接收提交的老师 |
| `originalFileName` | `String` | 是 | 无 | 无 | 学生提交时填写或当前文件的原始文件名 |
| `submittedFileName` | `String` | 是 | 无 | 无 | 系统生成的提交文件名，格式为 `学生用户名_文件名` |
| `language` | `String` | 是 | 无 | 枚举：`python` / `cpp` / `nodejs` | 提交代码语言 |
| `content` | `String` | 否 | `''` | 无 | 提交的代码内容 |
| `createdAt` | `Date` | 是 | 自动生成 | 无 | 提交时间 |
| `updatedAt` | `Date` | 是 | 自动生成 | 无 | 更新时间 |

访问规则：

| 操作 | 限制 |
| --- | --- |
| 创建提交 | 需要 `canSubmitCode` 权限，且当前用户角色必须是 `student` |
| 查看自己的提交 | 需要 `canSubmitCode` 权限，只返回 `student` 为当前用户的记录 |
| 查看收到的提交 | 需要 `canReviewSubmissions` 权限，只返回 `teacher` 为当前用户的记录 |

## 7. 集合：systemsettings

用途：保存系统级配置。当前主要用于保存 AI API 配置。

模型文件：`backend/src/models/SystemSetting.js`

| 字段 | 类型 | 必填 | 默认值 | 约束 / 索引 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `_id` | `ObjectId` | 是 | 自动生成 | 主键 | 配置记录 ID |
| `key` | `String` | 是 | 无 | `unique: true`，`trim: true` | 配置项名称 |
| `value` | `Mixed` | 否 | `{}` | 无 | 配置内容 |
| `createdAt` | `Date` | 是 | 自动生成 | 无 | 创建时间 |
| `updatedAt` | `Date` | 是 | 自动生成 | 无 | 更新时间 |

### 7.1 AI 配置项

当前 AI 配置使用：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `key` | `String` | 固定为 `aiTutor` |
| `value.provider` | `String` | 服务商，例如 `siliconflow`、`openai`、`custom` |
| `value.apiBaseUrl` | `String` | API 基础地址，例如 `https://api.siliconflow.cn/v1` |
| `value.model` | `String` | 模型名称，例如 `deepseek-ai/DeepSeek-V3` |
| `value.apiKey` | `String` | 明文 API 密钥，仅后端调用 AI 服务时使用 |
| `value.apiKeyPreview` | `String` | 密钥预览，例如 `sk-...abcd`，用于前端显示配置状态 |

安全说明：

| 项 | 说明 |
| --- | --- |
| 存储方式 | 按当前需求明文存储在 `systemsettings.value.apiKey` |
| 前端可见内容 | 只能看到 `apiKeyPreview`，不能读取完整 API Key |
| 生产建议 | 限制数据库访问权限，避免把数据库备份和日志公开 |

## 8. 实体关系

```text
User 1 ── N File
  一个用户可以拥有多个个人代码文件

User(student) 1 ── N Submission
  一个学生可以产生多条提交记录

User(teacher) 1 ── N Submission
  一个老师可以收到多条提交记录

SystemSetting
  独立系统配置，不属于任何单个用户
```

## 9. API 数据对象

本节整理主要请求体和响应体中的数据对象。

### 9.1 登录

接口：`POST /api/auth/login`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | `String` | 是 | 用户名，会 trim 并转小写 |
| `password` | `String` | 是 | 明文密码，仅用于登录校验 |

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 固定为 `true` |
| `token` | `String` | JWT，后续请求放入 `Authorization: Bearer <token>` |
| `user` | `UserPublic` | 当前登录用户公开信息 |

### 9.2 当前用户

接口：`GET /api/auth/me`

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 固定为 `true` |
| `user` | `UserPublic` | 当前登录用户公开信息 |

### 9.3 运行代码

接口：`POST /api/execute`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | `String` | 是 | 待执行代码 |
| `language` | `String` | 是 | `python` / `cpp` / `nodejs` |
| `stdin` | `String` | 否 | 标准输入内容，默认空字符串 |

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 程序退出码是否为 0 |
| `output` | `String` | 标准输出 stdout |
| `error` | `String` | 标准错误 stderr 或执行异常信息 |
| `exitCode` | `Number` | 退出码；超时为 `124`，后端执行异常为 `-1` |
| `executionId` | `String` | 单次执行 ID，用于排查日志 |

### 9.4 生成 AI 教学指导

接口：`POST /api/execute/guidance`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `code` | `String` | 是 | 最近一次运行失败的代码 |
| `language` | `String` | 是 | `python` / `cpp` / `nodejs` |
| `stdin` | `String` | 否 | 最近一次运行使用的标准输入 |
| `executionResult.success` | `Boolean` | 是 | 必须为 `false` |
| `executionResult.output` | `String` | 否 | 最近一次运行的 stdout |
| `executionResult.error` | `String` | 否 | 最近一次运行的 stderr 或错误信息 |
| `executionResult.exitCode` | `Number` | 否 | 最近一次运行的退出码 |

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 固定为 `true` |
| `guidance` | `String` | AI 返回的教学指导文本 |

### 9.5 文件保存 / 更新

接口：

- `POST /api/files`
- `PUT /api/files/:id`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | `String` | 是 | 文件名，会 trim |
| `content` | `String` | 否 | 文件内容 |
| `language` | `String` | 是 | `python` / `cpp` / `nodejs` |

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 固定为 `true` |
| `file` | `File` | 保存后的文件对象 |

### 9.6 用户创建 / 更新

创建接口：`POST /api/users`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `username` | `String` | 是 | 用户名，会 trim 并转小写 |
| `password` | `String` | 是 | 初始密码 |
| `name` | `String` | 否 | 显示名，默认使用用户名 |
| `role` | `String` | 否 | `teacher` / `student`，默认学生 |
| `permissions` | `Object` | 否 | 权限覆盖项，会与角色默认权限合并 |

更新接口：`PUT /api/users/:id`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `name` | `String` | 否 | 显示名 |
| `role` | `String` | 否 | `teacher` / `student` |
| `isActive` | `Boolean` | 否 | 是否启用账号 |
| `permissions` | `Object` | 否 | 权限覆盖项 |
| `password` | `String` | 否 | 新密码；为空则不修改 |

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 固定为 `true` |
| `user` | `UserPublic` | 更新后的用户公开信息 |

### 9.7 提交代码

接口：`POST /api/submissions`

请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `teacherId` | `ObjectId` | 是 | 接收提交的老师 ID |
| `fileName` | `String` | 是 | 提交文件名，会清理非法字符 |
| `language` | `String` | 是 | `python` / `cpp` / `nodejs` |
| `content` | `String` | 否 | 提交代码内容 |

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 固定为 `true` |
| `submission` | `Submission` | 新建的提交记录 |

### 9.8 AI 设置

接口：

- `GET /api/ai-settings`
- `PUT /api/ai-settings`
- `DELETE /api/ai-settings`

权限要求：需要 `canManageAiSettings`。

`PUT /api/ai-settings` 请求体：

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `provider` | `String` | 是 | `siliconflow` / `openai` / `custom` |
| `apiBaseUrl` | `String` | 是 | OpenAI-compatible API 基础地址 |
| `model` | `String` | 是 | 模型名称 |
| `apiKey` | `String` | 首次配置必填 | API Key；后续留空表示保持原密钥 |

成功响应：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `Boolean` | 固定为 `true` |
| `settings.configured` | `Boolean` | 是否已配置 API Key |
| `settings.apiKeyPreview` | `String` | API Key 预览，不包含完整密钥 |
| `settings.provider` | `String` | 当前服务商 |
| `settings.apiBaseUrl` | `String` | 当前 API 地址 |
| `settings.model` | `String` | 当前模型 |
| `settings.updatedAt` | `Date` | 更新时间 |

## 10. 关键环境变量
